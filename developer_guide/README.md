# Developer Roadmap: Zero to Developer in 3 Months
**Philosophy: "Reverse Engineering"**

Becoming a developer in 3 months is ambitious, but you have a significant advantage: **you already own a professional-grade codebase.** Instead of starting from blank screens, you will learn by modifying, breaking, and fixing your existing code.

## 📅 The 12-Week Timeline

### **Month 1: The "Frontend" Mechanic**
*   **Focus:** HTML, CSS, Tailwind, Layouts.
*   **Goal:** Modify the *look* of your website without breaking the logic.
*   **Key Files:** `src/pages/Index.tsx`, `src/components/ui/button.tsx`.
*   [Go to Month 1 Guide](./01_Month_1_Frontend.md)

### **Month 2: The "Logic" Engineer**
*   **Focus:** JavaScript, React Hooks (State, Effects), Functions.
*   **Goal:** Make things *do* something when interacted with.
*   **Key Files:** `src/lib/constants.ts`, `src/pages/dashboard/TakeExam.tsx`.
*   [Go to Month 2 Guide](./02_Month_2_Logic.md)

### **Month 3: The "Backend" Architect**
*   **Focus:** Database (SQL), APIs, Security (RLS), Deployment.
*   **Goal:** Understand where the data lives and how to protect it.
*   **Key Tools:** Supabase Dashboard, GitHub, Vercel.
*   [Go to Month 3 Guide](./03_Month_3_Backend.md)

## 🛠 Prerequisites (Set these up immediately)
1.  **VS Code:** Your code editor.
2.  **Node.js:** The environment to run your code (`npm run dev`).
3.  **Git:** To save your history.
4.  **Browser DevTools:** Right-click -> Inspect is your best friend.

## 🛑 Golden Rules
1.  **Read the Red Text:** Error messages tell you exactly what is wrong. Don't ignore them.
2.  **Change One Thing at a Time:** If you change 10 things and it breaks, you won't know why.
3.  **Google is Part of the Job:** "How to center div tailwind" is a valid search for a senior engineer.
